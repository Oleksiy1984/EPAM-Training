Данный проект дан в качестве примера. Сервлеты не должны содержать HTML разметки.
Для формирования представления используйте JSP (см. проект Practice9Jsp)

1. Сконфигурировать Tomcat в Eclipse, если он еще не сконфигурирован.

2. Подключить к проекту Tomcat, если он еще не подключен.

3. Для запуска Apache Derby возможно потребуется раздать права JVM прослушивать порт 1527 (порт по умолчанию Derby, запущенного в режиме client-server).
Чтобы это сделать, для той JRE, которая выполняет скрипт запуска Apache Derby Server (см. в настройках проекта/IDE) добавить следующие строки в файл
JRE_HOME\lib\security\java.policy:
 
grant {
    permission java.net.SocketPermission "localhost:1527", "listen";    
};

Замечание: возможно потребуется запускать редактор с помощью которого вы будете делать правки в этом файле от имени администратора системы.

4. Для того, чтобы Tomcat правильно разбирал имена и значения параметров запросов, которые содержат кириллицу, необходимо, чтобы сервер использовал при разборе кодировку, содержащую кириллицу.
Как правило, символы, которые не являются символами ASCII, браузер передает на сервер в кодированном виде используя т.н. percent-encoding.
Для разбора же имен и значений параметров Tomcat по умолчанию использует кодировку Latin-1.
В качестве кодировки целесообразно указать UTF-8.

Для POST запросов можно определить фильтр, который фильтруя все запросы устанавливает в них кодировку UTF-8.

Для GET запросов необходимо делать правки на серверной стороне. Если в Eclipse сконфигурирован Tomcat (например Tomcat 7), то нужно отредактировать данный файл:
Servers\Tomcat v7.0 Server at localhost-config\server.xml

Например, если изначально былок так:
<Connector connectionTimeout="20000" port="8080" protocol="HTTP/1.1" redirectPort="8443"/>

То после правки должно быть так:
<Connector connectionTimeout="20000" port="8080" protocol="HTTP/1.1" redirectPort="8443" URIEncoding="UTF-8" />
